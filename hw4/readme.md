# 多項式求根專案筆記

這次作業主要是寫出二、三次方程的公式解，並思考五次以上沒有公式時該怎麼辦。

## 1. 二次與三次方程 (root2 & root3)
### 實作思路
* **核心公式**：直接套用數學公式算。二次用判別式，三次參考維基百科的卡爾丹公式。
* **一定要用 `cmath`**：因為公式裡常有開根號，如果判別式是負的，普通的 `math.sqrt` 會噴錯。用 `cmath` 才能處理複數根 $j$。
* **誤差處理**：電腦算浮點數會有極小誤差（例如 $10^{-16}$），所以驗證時不能直接用 `== 0`，要用 `cmath.isclose` 看是不是夠接近 0。

## 2. 高次方程 (n >= 5) 的思考
### 為什麼不能套公式？
* **限制**：讀了資料發現有「伽羅瓦定理」，數學上已經證明五次以上的方程式沒有一般的公式解。
* **策略**：沒公式，得改用「數值搜尋」的方法。

### 爬山演算法 (Hill Climbing) 邏輯
這就像是在複數平面上玩「冷熱遊戲」來找谷底：
1. **找谷底**：把 $|f(x)|$ 的值當成高度，高度是 0 的地方就是根。
2. **隨機跳點**：先隨便猜一個點 $x$ 當起點。
3. **往低處走**：檢查這個點的四周（實部與虛部），哪邊的值變小了就往哪邊移。
4. **越縮越小**：如果四周都比現在高，就縮小腳步（步長）再找細一點，直到值夠接近 0 為止。
